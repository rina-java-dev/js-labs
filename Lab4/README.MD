## Ход работы

### Шаг 1. Создание класса `Item`

Сначала был создан класс `Item`, который описывает обычный предмет инвентаря.

Класс содержит три поля:

- `name` — название предмета
- `weight` — вес предмета
- `rarity` — редкость предмета

Также были реализованы два метода:

- `getInfo()` — возвращает строку с полной информацией о предмете
- `setWeight(newWeight)` — изменяет вес предмета

### Пример:

const sword = new Item("Steel Sword", 3.5, "rare");
console.log(sword.getInfo());
sword.setWeight(4.0);

### Шаг 2. Создание класса Weapon

После этого был создан класс Weapon, который расширяет класс Item.

Он наследует все свойства и методы Item, а также добавляет два новых поля:

damage — урон оружия
durability — прочность оружия

Кроме того, были добавлены новые методы:

use() — уменьшает durability на 10, если прочность больше 0
repair() — восстанавливает durability до 100

Для вызова конструктора родительского класса используется super(...).

### Пример:
const bow = new Weapon("Longbow", 2.0, "uncommon", 15, 100);
console.log(bow.getInfo());
bow.use();
console.log(bow.durability);
bow.repair();

### Шаг 4. Использование опциональной цепочки

В программе была использована опциональная цепочка ?..

Она нужна для безопасного доступа к свойствам объектов.
Если объект равен null или undefined, ошибка не возникает, а результатом будет undefined.

### Пример:
const inventory = {
    mainWeapon: bow,
    backupWeapon: null
};

console.log(inventory.mainWeapon?.damage);
console.log(inventory.backupWeapon?.damage);
console.log(inventory.owner?.nickname);

Здесь:

inventory.mainWeapon?.damage вернет значение урона
inventory.backupWeapon?.damage не вызовет ошибку, а вернет undefined
inventory.owner?.nickname тоже безопасно вернет undefined


### Шаг 5. Переписывание через функции-конструкторы

После реализации через class аналогичная логика была переписана через функции-конструкторы.

Были созданы:

функция-конструктор Item
функция-конструктор Weapon

Для наследования использовались:

Item.call(this, ...) — чтобы вызвать родительский конструктор
Object.create(Item.prototype) — чтобы унаследовать методы родителя
Почему методы записаны в prototype

Методы были вынесены в prototype, потому что тогда они создаются один раз и используются всеми объектами.
Если записывать методы прямо внутри конструктора, то для каждого объекта будет создаваться новая копия функции, а это менее эффективно.

### Контрольные вопросы

1. Какое значение имеет this в методах класса?

this в методах класса указывает на конкретный объект, у которого вызван этот метод.

 
2. Как работает модификатор доступа # в JavaScript?

Символ # используется для создания приватных полей и методов класса.

Приватные поля доступны только внутри самого класса и недоступны извне объекта

Такой механизм нужен, чтобы скрывать внутренние данные объекта и защищать их от прямого изменения снаружи.

3. В чем разница между классами и функциями-конструкторами?

Классы и функции-конструкторы решают похожую задачу: позволяют создавать объекты по шаблону.
Но между ними есть различия.

Классы:
имеют более современный и удобный синтаксис
используют extends для наследования
используют super() для вызова конструктора родителя

Функции-конструкторы:
это более старый способ создания объектов
наследование реализуется вручную через call() и prototype
